# 交互式光学实验室 - 产品开发方案 (v0.2)

**最终目标:** 开发一个功能强大、物理精确、界面美观、交互流畅、易于使用的跨平台光学仿真工具，适用于教育教学、科研初步验证、光学设计演示等场景，并最终以 Web 应用或独立应用的形式发布。

**核心原则:**

*   追求完美、注重细节
*   用户体验优先
*   物理准确性与性能平衡
*   分阶段迭代开发
*   **专业界面对标:** 借鉴成熟 CAD/CAE 软件（如 SolidWorks）的界面布局和交互逻辑，提升专业感和操作效率。

**开发阶段划分:**

**第一阶段：核心功能完善与体验优化 (当前进行中及近期)**

*   **目标:** 巩固现有功能，修复已知问题，提升核心交互体验，为后续扩展打好基础。
*   **任务列表:**
    *   **[已完成]** 基本元件库（光源、镜、片、屏、块、缝、栅、偏振、波片、AOM、光纤、探测器等）。
    *   **[已完成]** 核心物理模拟（反射、折射、色散、衍射基础、偏振基础、干涉基础）。
    *   **[已完成]** 元件选择、移动、旋转、删除。
    *   **[已完成]** 属性检查器面板（标签页结构）。
    *   **[已完成]** 画布缩放、平移。
    *   **[已完成]** 场景本地保存/加载/管理（通过“场景”标签页）。
    *   **[已完成]** 模式切换（光线追踪 vs 透镜成像）。
    *   **[进行中]** 基础 UI 美化（字体、颜色、布局、顶部菜单图标化、撤销/重做图标）。
    *   **[进行中]** 全局设置面板（通过“设置”标签页，包括网格、光线数、弹射数、强度阈值等）。
    *   **[待办]** **撤销/重做 (Undo/Redo):** 实现完整的操作历史记录和撤销/重做功能。 *(注：基础框架已搭建，但需确保覆盖所有操作并进行测试)*
    *   **[待办]** 修复已知 Bug（例如，检查特定元件交互问题、UI 显示问题、撤销/重做覆盖范围等）。
    *   **[待办]** **对齐辅助线:** 拖动元件时，显示与其他元件中心或边缘对齐的参考线。
    *   **[待办]** **元件吸附:** 靠近对齐线或网格点时，元件自动吸附。
    *   **[待办]** **多选与成组:** 实现按住 Shift/Ctrl 点击多选元件，以及将多个元件组合成一个整体进行移动/旋转的功能。
    *   **[待办]** **左侧工具栏图标化:** 采用图标为主、文字提示为辅的工具按钮，优化布局和分组。
    *   **[待办]** **右侧检查器结构化:** 改进属性面板布局（分组、视觉元素），考虑引入组件树视图。
    *   **[待办]** **浅色主题选项:** 开发并允许用户切换到更接近标准 CAD 软件的浅色界面主题。
    *   **[待办]** **自定义 Tooltip:** 替换浏览器默认 title 提示，使用更美观、信息更丰富的自定义提示框。
    *   **[待办]** **用户指南完善:** 将生成的文本指南集成到“帮助”菜单下的模态框中。
    *   **[待办]** **登录/注册功能:** 实现基本的（前端）用户登录和注册界面逻辑（后端待定）。
    *   **[待办]** **导入/导出健壮性:** 增加对导入文件格式的校验，优化导出数据的完整性，处理默认保存位置。

**第二阶段：物理模型深化与元件库扩展**

*   **目标:** 提高模拟的物理精度，增加更多常用和高级的光学元件。
*   **任务列表:**
    *   **高斯光束完善:**
        *   精确追踪光束宽度和曲率半径。
        *   实现高斯光束通过透镜、反射镜后的变换（ABCD 矩阵或直接计算）。
        *   可视化高斯光束轮廓（例如用半透明填充或多条代表强度的线）。
    *   **衍射模型增强:**
        *   为光阑/多缝添加基于惠更斯原理或傅里叶光学的衍射计算，在屏幕上显示更真实的衍射图样（包括单缝衍射包络）。
        *   考虑圆形光瞳的艾里斑衍射。
    *   **干涉模型增强:**
        *   模拟部分相干光源对干涉条纹对比度的影响。
        *   实现法布里-珀罗干涉仪、马赫-曾德干涉仪等典型干涉仪的模拟。
    *   **偏振模型深化:**
        *   实现菲涅尔方程，计算不同偏振态在界面上的反射率和透射率。
        *   模拟布儒斯特角现象。
        *   精确模拟波片对不同偏振态（包括圆偏振、椭圆偏振）的作用 (Jones 矩阵)。
    *   **透镜像差模拟:**
        *   加入更具体的球差、彗差、像散、场曲、畸变模型（可能需要更复杂的透镜定义）。
        *   可视化像差对成像点或屏幕图样的影响。
    *   **新元件添加:**
        *   厚透镜 (几何光学与物理光学模型)。
        *   渐变折射率 (GRIN) 元件。
        *   光隔离器。
        *   非球面透镜/反射镜。
        *   （更后期）液晶调制器、空间光调制器 (SLM) (需要新的交互方式)。
    *   **材质库:** 内建常用光学玻璃（肖特、康宁等）和自定义材料的数据库（折射率模型如 Sellmeier、热光系数等）。

**第三阶段：高级功能与产品化**

*   **目标:** 实现用户账户系统、云同步、协作、导出等高级功能，将其包装成完整的 Web 应用或跨平台应用。
*   **任务列表:**
    *   **后端开发:**
        *   选择技术栈 (Node.js/Python/Go 等)。
        *   设计并实现数据库模型（用户、场景等）。
        *   开发用户注册、登录、认证 (Token/Session) API。
        *   开发场景数据 CRUD (创建、读取、更新、删除) API。
        *   部署后端服务。
    *   **前端集成:**
        *   实现登录/注册模态框与后端 API 的交互。
        *   实现用户状态管理和持久化。
        *   实现“保存到云端”、“加载云端场景”、“我的场景”列表功能。
    *   **单位与标尺:**
        *   引入物理单位（mm, μm, nm）的概念。
        *   设定像素与物理单位的比例尺，并允许用户调整。
        *   在界面上显示标尺或坐标轴。
    *   **结果导出:**
        *   导出画布为 PNG/JPG/SVG 图像。
        *   导出屏幕/光度计数据为 CSV 或文本文件。
    *   **协作功能 (远期):**
        *   允许多个用户实时或异步协作编辑同一个场景。
    *   **性能优化:**
        *   代码优化，减少不必要的计算和重绘。
        *   Web Workers：将复杂计算（如光追）放到后台线程，避免 UI 卡顿。
        *   WebGL：考虑使用 WebGL 进行硬件加速渲染，尤其是对于大量光线或复杂的可视化。
    *   **跨平台打包 (远期):**
        *   使用 Electron, Tauri 或类似技术将 Web 应用打包成本地桌面应用 (Windows, macOS, Linux)。
        *   考虑 PWA (Progressive Web App) 以提供类似应用的体验。
    *   **命令管理器/Ribbon 界面 (远期):** 探索实现类似 Office/SolidWorks 的 Ribbon 界面，将功能按任务分组，提供更丰富的可视化工具访问。
    *   **高级可视化:** 光束轮廓（高斯）、波前、偏振态（庞加莱球或椭圆）、点列图、MTF 曲线等。
    *   **脚本与自动化:** 提供 API 或脚本接口（JavaScript/Python），允许用户进行参数扫描、优化、自动化测试和自定义分析。
    *   **互操作性:** 支持导入/导出标准光学设计文件格式（如 ZEMAX .zmx/.zos - 基础部分）、几何格式（STEP/IGES）或数据格式（CSV、羽毛）。

**第四阶段：持续迭代与维护**

*   **目标:** 根据用户反馈进行优化，修复 Bug，添加新功能，保持技术更新。
*   **任务列表:**
    *   建立用户反馈渠道。
    *   定期进行 Bug 修复和性能调优。
    *   根据需求和技术发展，持续添加新的光学元件和物理模型。
    *   更新用户文档和教程。
    *   维护后端服务和数据库。

**时间线与优先级 (初步建议):**

1.  **近期 (现在 -> 几周):** 完成第一阶段剩余的 **UI 优化**（左侧工具栏图标化、右侧检查器结构化初步、浅色主题）、**撤销/重做**功能完善、**对齐/吸附**、**Bug 修复**。
2.  **中期 (几周 -> 几个月):** 开始第二阶段，优先完善**高斯光束**和**衍射模型**。开始添加**厚透镜**。同时进行**登录/注册**前端逻辑和基础**后端**搭建。
3.  **长期 (几个月 -> 更长):** 逐步推进第二阶段和第三阶段的功能。

**成功关键因素:**

*   代码质量与可维护性
*   物理准确性
*   性能意识
*   用户反馈
*   迭代开发
